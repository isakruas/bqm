   Olá, sou Sibele, sua assistente virtual.

Bem vindo(a) ao Manual de Utilização do Banco de Questões de Matemática (BQM), aqui você encontrará informações que lhe permitirão entender a maneira em que esta plataforma funciona.

Guia de edição

A edição das questões é como a edição de qualquer texto, o que diferencia o uso do editor desta plataforma são os recursos adicionais de formação de texto que ela permite; ele foi otimizado para tornar o processo de escrever fórmulas matemáticas mais simples. Admito que o editor não parece muito bonito e para alguns pode ser estranho usá-lo no início, mas garanto que, quando você começar a praticar, o processo se torna simples e agradável. Pretendemos deixar o nosso editor o mais intuitivo possível, para isso, procuramos desenvolver mecanismos que o ajudem no processo de editoração, no entanto, algumas melhorias podem demorar alguns meses para serem implementadas, entretanto, com o crescimento desta plataforma e com a ajuda de mais colaboradores, acreditamos que as melhorias serão disponibilizadas em intervalos de tempo cada vez mais curtos.

O editor desta plataforma é sensível à linguagem de programação Tex (https://pt.wikipedia.org/wiki/TeX) e a linguagem de marcação HTML (https://pt.wikipedia.org/wiki/HTML), ou seja, é possivel combinar os recursos do Tex com os recursos do HTML no processo de escrita das questões. Pode parecer complicado, mas não se preocupe, estou aqui para lhe mostrar que não é um bicho de sete cabeças utilizar os recursos do HTML e Tex quando se estiver escrevendo as questões. Irei me ater aqui em recursos básicos que serão usados com mais frequências por vocês, quando estiverem escrevendo as questões. Para utilização de recursos mais avançados, recomendo que busquem materiais especializados que irão aprofundar mais sobre a utilização destas linguagens.

Comandos em Tex

Há duas formas de se inserir um comando em Tex no editor desta plataforma, a primeira, é inseri-los desta forma: \( comando \) . Ou seja, quando o nosso editor se deparar com o comando \( ele irar entender que dali em diante, o texto escrito estará na linguagem Tex, e quando ele se deparar com \) ele irá entender que ali se encerra o comando em Tex. Utilizando esta simbologia, o editor irá entender que se trata de um comando em Tex e que ele deverá exibir o resultado deste comando na mesma linha em que ele foi inserido. Calma, pode parecer confuso, mas vamos analisar um exemplo para entender melhor.

Exemplo:

A função quadrática é uma função \( f : \mathbb{R} \mapsto \mathbb{R} \), definida como \( f(x) = ax^2 + bx + c \), com a, b e c números reais e \( a \ne 0 \).

Ao interpretar este texto, o editor irá produzir como resultado:

A função quadrática é uma função \( f : \mathbb{R} \mapsto \mathbb{R} \), definida como \( f(x) = ax^2 + bx + c \), com a, b e c números reais e \( a \ne 0 \).

Perceba que o comando em Tex é convertido para uma notação matemática com um aspecto visual mais agradável, asim como que este comando foi interpretado na mesma linha em que ele foi declarado. Ou seja, sempre que desejar inserir um comando escrito em Tex para que ele seja interpretado na mesma linha em que foi declarado, basta colocá-lo dentro de \( comando \)

A segunda forma de se declarar um comando em Tex destro do editor desta plataforma é usá-lo dentro das tags \[ comando \] , perceba que há uma semelhança ao comando apresentado acima, entretanto, o resultado que o editor irá retornar será diferente; quando os comandos são declarados entre \[ comando \] , o editor irá entender que trata-se em um comando em Tex, e que ele deve exibir o resultado uma linha abaixo da qual o comando foi declarada. Para que facilite a compreensão, e que você possa analisar a diferença entre este comando e o anterior, observe o resultado do exemplo a seguir.

Exemplo:

A função quadrática é uma função \[ f : \mathbb{R} \mapsto \mathbb{R} \], definida como \[ f(x) = ax^2 + bx + c \], com a, b e c números reais e \[ a \ne 0 \].

Ao interpretar este texto, o editor irá produzir como resultado:

A função quadrática é uma função \[ f : \mathbb{R} \mapsto \mathbb{R} \], definida como \[ f(x) = ax^2 + bx + c \], com a, b e c números reais e \[ a \ne 0 \].

Perceba que o resultado é semelhante ao apresentado anteriormente, quando consideramos o aspecto visual, porém, a forma em que o editor irá dispor os resultados é diferente; ele irá pular uma linha para exibir o resultado do comando apresentado. A escolha de um comando ou outro é subjetiva, ou seja, sinta-se livre para escolher uma ou a outra. Recomendamos que escolha a opção que melhor dispor de um resultado final, para que o enunciado, resolução ou observações possuam um aspecto visual agradável. Para mais exemplos de comandos, consulte a seção Fórmulas em Tex essenciais.

Comandos em HTML

Existem uma variedade de comandos em HTML, cada um com uma finalidade bem definida, não é meu objetivo lhe falar todas as propriedades desta linguagem, mas somente o básico para que possa utilizar nesta plataforma. Os comandos que iremos explorar são os oriundos das tags i, b, u, br, ul, e ol. Não se assuste, pode parecer complicado mas garanto, é mais simples do que você imagina!

Quando se deseja colocar alguma palavra em itálico, utilizamos a tag i do HTML, e a declaramos assim: <i> texto escrito em itálico </i>, e como resultado, o editor exibirá: texto escrito em itálico.

Quando se deseja colocar alguma palavra em negrito, utilizamos a tag b do HTML, e a declaramos assim: <b> texto escrito em negrito </b>, e como resultado, o editor exibirá: texto escrito em negrito.

Quando se deseja sublinhar alguma palavra, utilizamos a tag u do HTML, e a declaramos assim: <u> texto sublinhado </u>, e como resultado, o editor exibirá: texto sublinhado.

Quando se deseja criar uma quebra de linha no texto, utilizamos a tag br do HTML, e a declaramos assim: este é um exemplo de <br /> quebra de linha, e como resultado, o editor exibirá:
este é um exemplo de
quebra de linha.

Quando se deseja criar uma lista desordenada, ou lista não numerada, utilizamos a tag ul do HTML, e a declaramos assim: 

<ul>
  <li>São Paulo<\li>
  <li>Rio de Janeiro<\li>
  <li>Mato Grosso<\li>
</ul>
, e como resultado, o editor exibirá:
  • São Paulo
  • Rio de Janeiro
  • Mato Grosso
.

Quando se deseja criar uma lista ordenada, ou lista numerada, utilizamos a tag ol do HTML, e a declaramos assim: 

<ol>
  <li>São Paulo<\li>
  <li>Rio de Janeiro<\li>
  <li>Mato Grosso<\li>
</ol>
, e como resultado, o editor exibirá:
  1. São Paulo
  2. Rio de Janeiro
  3. Mato Grosso
.
